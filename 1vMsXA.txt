		<h1>HTML
			<span>Elements</span>
		</h1>
		<p class="summary">In this tutorial you will learn about HTML tags and elements.</p>
		<h2>HTML Element Syntax</h2>
		<p>An HTML element is an individual component of an HTML document. It represents semantics, or meaning. For example, the
			<code>title</code>element represents the title of the document.</p>
		<p class="space">Most HTML elements are written with a
			<em>start tag</em>(or opening tag) and an
			<em>end tag</em>(or closing tag), with content in between. Elements can also contain attributes that defines its additional properties. For example, a paragraph, which is represented by the
			<code>p</code>element, would be written as:</p>
		<div class="space">
			<img width="560" src="../lib/images/html-element.png" alt="HTML Element Syntax Illustration" />
		</div>
		<p>We will learn about the HTML attributes in the
			<a href="html-attributes.php">next chapter</a>.</p>

		<h2>HTML Tags Vs Elements</h2>
		<p>Technically, an HTML element is the collection of start tag, its attributes, an end tag and everything in between. On the other hand an HTML tag (either opening or closing) is used to mark the start or end of an element, as you can see in the above illustration.</p>
		<p>However, in common usage the terms HTML element and HTML tag are interchangeable i.e. a tag is an element is a tag. For simplicity's sake of this website, the terms "tag" and "element" are used to mean the same thing — as it will define something on your web page.</p>
		<hr />
		<h2>Case Insensitivity in HTML Tags and Attributes</h2>
		<p>In HTML, tag and attribute names are not case-sensitive (but most attribute values are case-sensitive). It means the tag
			<code>&lt;P&gt;</code>, and the tag
			<code>&lt;p&gt;</code>defines the same thing in HTML which is a paragraph.</p>
		<p>But in
			<a href="../xhtml-tutorial.php">XHTML</a>they are case-sensitive and the tag
			<code>&lt;P&gt;</code>is different from the tag
			<code>&lt;p&gt;</code>.</p>

		<h2 id="empty-elements">Empty HTML Elements</h2>
		<p>Empty elements (also called self-closing or void elements) are not container tags — that means, you can not write
			<code>&lt;hr&gt;
				<i>some content</i>&lt;/hr&gt;</code>or
			<code>&lt;br&gt;
				<i>some content</i>&lt;/br&gt;</code>.</p>
		<p>A typical example of an empty element, is the
			<code>&lt;br&gt;</code>element, which represents a line break. Some other common empty elements are
			<code>&lt;img&gt;</code>,
			<code>&lt;input&gt;</code>,
			<code>&lt;link&gt;</code>,
			<code>&lt;meta&gt;</code>,
			<code>&lt;hr&gt;</code>, etc.</p>

		<h2>HTML Elements Types</h2>
		<p>Elements can be placed in two distinct groups:
			<em>
				<a href="../css-tutorial/css-visual-formatting.php#block-level">block level</a>
			</em>and
			<em>
				<a href="../css-tutorial/css-visual-formatting.php#inline-level">inline level</a>
			</em>elements. The former make up the document's structure, while the latter dress up the contents of a block.</p>
		<p>Also, a block element occupies 100% of the available width and it is rendered with a line break before and after. Whereas, an inline element will take up only as much space as it needs.</p>
		<p>The most commonly used block-level elements are
			<code>&lt;div&gt;</code>,
			<code>&lt;p&gt;</code>,
			<code>&lt;h1&gt;</code>through
			<code>&lt;h6&gt;</code>,
			<code>&lt;form&gt;</code>,
			<code>&lt;ol&gt;</code>,
			<code>&lt;ul&gt;</code>,
			<code>&lt;li&gt;</code>, and so on. Whereas, the commonly used inline-level elements are
			<code>&lt;img&gt;</code>,
			<code>&lt;a&gt;</code>,
			<code>&lt;span&gt;</code>,
			<code>&lt;strong&gt;</code>,
			<code>&lt;b&gt;</code>,
			<code>&lt;em&gt;</code>,
			<code>&lt;i&gt;</code>,
			<code>&lt;code&gt;</code>,
			<code>&lt;input&gt;</code>,
			<code>&lt;button&gt;</code>, etc.</p>
		<p>You will learn about these elements in detail in upcoming chapters.</p>
